#### Design Approach

**StreamingTranscriptionService State Machine:**
Replace boolean `isProcessing`/`isFinishing` flags with explicit states:
- `Idle` -> `Recording` (on startProcessing)
- `Recording` -> `Draining` (on finishProcessing -- stops accepting new chunks, continues processing queue)
- `Draining` -> `Complete` (queue empty or timeout)
- Any -> `Idle` (on reset/cleanup)

TimerModal must await the `Draining -> Complete` transition before closing modal and delivering results.

**Tiered Memory Profiles:**
```
Tier 1 (Constrained): < 512MB available -> 100MB limit, 3s chunks, queue of 2
Tier 2 (Standard): 512MB-2GB available -> 200MB limit, 5s chunks, queue of 3
Tier 3 (Generous): > 2GB available -> 400MB limit, 10s chunks, queue of 5
```
Detection: `performance.memory` (Chromium) -> `navigator.deviceMemory` -> conservative fallback.

**Lazy Validation:**
Remove `validateApiKeys()` from `onload()`. Add `ensureValidated()` method to AIAdapter that validates on first use and caches the result.

**OpenRouter Adapter:**
Extend `AIAdapter` following OpenAI/Groq pattern. Base URL: `https://openrouter.ai/api/v1`. Curated default model list + custom model ID text field for power users. Serves as both `transcriptionProvider` and `postProcessingProvider`.

#### Key Decisions
| Decision | Options | Recommendation | Rationale |
|----------|---------|----------------|-----------|
| OpenRouter model list | A) Static curated B) Dynamic API fetch C) Static + custom ID | C) Static + custom ID | Avoids staleness while keeping simplicity; OpenRouter has hundreds of models |
| Memory calibration | A) Flat values (256/512MB) B) Tiered profiles based on measured memory | B) Tiered profiles | Handles device spectrum correctly (2GB Android vs 16GB desktop) |
| StreamingTranscriptionService state | A) Fix boolean ordering B) State machine refactor | B) State machine | Boolean flags are root cause of race conditions; explicit states prevent future bugs |
| Streaming audio save | A) Accumulate all chunks in memory B) Save chunks incrementally to disk | A) Accumulate in memory | Simpler; chunk data is already in memory during processing. Memory profiles constrain total size. |
| Console.log replacement | A) Just remove B) Replace with debug logger C) Build-time strip | A) Just remove | Obsidian policy prohibits console.log in production. No logging abstraction needed for plugin of this size. |
